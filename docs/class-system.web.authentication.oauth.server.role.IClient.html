Interface IClient

Represents a client application and must be implemented by any class that represents a third-party entity that will be making requests to protected resources while verified by the OAuth2 Provider.

From RFC 6749 - The OAuth 2.0 Authorization Framework:

CLIENT An application making protected resource requests on behalf of the resource owner and with its authorization. The term "client" does not imply any particular implementation characteristics (e.g., whether the application executes on a server, a desktop, or other devices).
